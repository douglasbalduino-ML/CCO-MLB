🔑 Git & SSH
❌
winget: comando não encontrado
O terminal retorna winget : O termo 'winget' não é reconhecido
Causa

O App Installer do Windows não está atualizado. O winget exige Windows 10 ≥ 1809 e App Installer atualizado.

Solução
Abra a Microsoft Store e busque App Installer — clique em Atualizar
Feche e reabra o PowerShell após a atualização
Alternativa: baixe o Git direto em git-scm.com/download/win
❌
ssh-keygen: Permission denied criando a chave
Could not open a connection to your authentication agent ou chave não aparece em ~/.ssh/
Causa

O serviço ssh-agent do Windows precisa estar em execução antes de adicionar a chave.

Solução
Abra o PowerShell como Administrador
Execute os comandos abaixo para ativar o serviço:
Set-Service ssh-agent -StartupType Automatic Start-Service ssh-agent
Feche e reabra o terminal normal, depois repita: ssh-add ~/.ssh/id_ed25519
⚠
GitHub rejeita a chave SSH: "Permission denied (publickey)"
git@github.com: Permission denied (publickey). após adicionar a chave
Causa mais comum

A chave foi adicionada ao GitHub mas a organização Meli exige SSO Authorization para a chave SSH.

Solução
Acesse github.com/settings/keys
Localize a chave recém-adicionada e clique em "Configure SSO"
Clique em "Authorize" ao lado da organização dbalduino_meli (ou sua org Meli)
Teste novamente: ssh -T git@github.com
⚠ Sem o SSO Authorization, o push/pull falha mesmo com a chave corretamente instalada no agent.
⚠
Git não encontra o repositório após clonar
fatal: not a git repository (or any of the parent directories)
Causa

O terminal está fora do diretório clonado. O VSCode não mudou automaticamente de pasta.

Solução
Menu File → Open Folder e selecione a pasta do repositório clonado
Ou no terminal: cd ~/workspace/cco-fm-workspace (ajuste o caminho)
🔐 Autenticação Claude Code
❌
Login falha com conta Gmail ou Mercado Pago
Página de login abre mas retorna erro ou fica em loop após autenticar
Causa

O Claude Code usa autenticação via Anthropic Console com SSO Meli. Somente contas @mercadolivre.com são aceitas.

Solução
Feche a janela do browser e volte ao VSCode
Clique novamente em "Sign in with Anthropic Console"
Na tela de login, clique em "Sign in with SSO" e informe o domínio mercadolivre.com
Use seuldap@mercadolivre.com (ex: psilva@mercadolivre.com)
⚠ Nunca use: @gmail.com · @hotmail.com · @mercadopago.com · conta pessoal Anthropic. Somente @mercadolivre.com funciona.
❌
Extensão Claude Code não aparece para instalar (VSCode)
Buscar "Claude Code" na aba Extensions não retorna resultados, ou o botão Install está bloqueado
Causa

Política corporativa pode bloquear instalação de extensões via Marketplace. Ou o VSCode está configurado para usar um Marketplace interno.

Solução
Tente instalar via linha de comando: code --install-extension anthropic.claude-code
Se bloqueado, abra um ticket de TI solicitando liberação da extensão anthropic.claude-code no Marketplace
Alternativa: use o Claude Code via terminal (claude CLI) sem a extensão VSCode
⚠
Sign in abre o browser mas o VSCode não confirma o login
Browser mostra "Authorization successful" mas o VSCode continua pedindo login
Causa

O callback de autenticação (vscode:// deep link) foi bloqueado pelo browser ou firewall corporativo.

Solução
Quando o browser exibir "Authorization successful", copie o código/URL exibido na página
Cole no campo que o VSCode mostra no terminal ou na caixa de diálogo de autenticação
Se o browser pergunta "Abrir VSCode?", clique em Permitir sempre
Se o popup não aparecer, tente desabilitar temporariamente o proxy corporativo
⚠
Licença não ativada / "No active subscription"
Claude Code instala e autentica mas mostra mensagem de plano ou sem acesso
Causa

A licença corporativa ainda não foi associada ao seu usuário no portal Azir.

Solução
Acesse o portal Azir interno da Meli e solicite a licença Claude Code
Após aprovação, reinicie o VSCode e faça login novamente com @mercadolivre.com
Se já tem licença aprovada mas continua o erro, execute no terminal: claude --logout e refaça o login
☁ Google Cloud / BigQuery
❌
gcloud: comando não encontrado após instalar o SDK
'gcloud' is not recognized as an internal or external command no PowerShell ou terminal do VSCode
Causa

O instalador do Cloud SDK adiciona o caminho ao PATH, mas a sessão do terminal estava aberta antes da instalação.

Solução
Feche completamente o VSCode (incluindo todos os terminais)
Reabra o VSCode — o novo PATH já é carregado automaticamente
Teste: gcloud --version
ℹ No Windows, o instalador do Cloud SDK cria um atalho "Google Cloud SDK Shell". Abra por ele na primeira vez para garantir que o PATH está correto.
❌
BigQuery retorna "Application Default Credentials not found"
Scripts Python falham com google.auth.exceptions.DefaultCredentialsError
Causa

Apenas gcloud auth login não é suficiente para bibliotecas Python. É necessário o Application Default Credentials (ADC).

Solução
Execute no terminal:
gcloud auth application-default login
Faça login com @mercadolivre.com no browser que abrir
Verifique: gcloud auth application-default print-access-token deve retornar um token
⚠
BigQuery: "Access Denied" em tabela meli-bi-data
403 Access Denied: Table meli-bi-data.WHOWNER.XXX
Causa

Seu usuário @mercadolivre.com precisa de permissão explícita nas tabelas do projeto meli-bi-data.

Solução
Confirme que está usando o billing project correto nas queries: ddme000162-8l63cxvebh8-furyid
Se ainda receber 403, solicite acesso ao dataset via CloudController ou abertura de ticket para o time de dados
Verifique se o gcloud config get-value project retorna o billing project correto
💻 VSCode
⚠
Git Bash não aparece na lista de terminais do VSCode
Ao abrir o dropdown de terminais, o Git Bash não é listado como opção
Causa

O VSCode precisa ser informado manualmente sobre o path do Git Bash após a instalação do Git.

Solução
Abra as configurações do VSCode: Ctrl+, → busque terminal.integrated.profiles.windows
Ou abra o arquivo settings.json (Ctrl+Shift+P → "Open User Settings JSON") e adicione:
"terminal.integrated.profiles.windows": { "Git Bash": { "source": "Git Bash" } }, "terminal.integrated.defaultProfile.windows": "Git Bash"
Feche e reabra o terminal no VSCode (Ctrl+`)
⚠
Terminal não abre no VSCode (ou abre mas fecha imediatamente)
O painel de terminal aparece vazio ou fecha sozinho após abrir com Ctrl+`
Causa

O shell padrão configurado não existe no sistema, ou o perfil do PowerShell tem um erro que causa falha na inicialização.

Solução
Acesse Ctrl+Shift+P → "Terminal: Select Default Profile"
Selecione PowerShell ou Command Prompt como fallback
Se ainda falhar, execute o VSCode com: code --disable-extensions para testar sem extensões
⚠
Extensão "Claude Code" não é encontrada na busca do Marketplace
Buscar por "Claude Code" no painel Extensions retorna apenas resultados irrelevantes
Causa

O Marketplace interno da Meli pode não incluir todas as extensões externas.

Solução
Tente buscar pelo ID exato: anthropic.claude-code
Ou instale via terminal: code --install-extension anthropic.claude-code
Se o Marketplace interno bloquear, acesse Ctrl+Shift+P → "Extensions: Install from VSIX" para instalar manualmente via arquivo .vsix
⚠
Claude responde mas não consegue ler/editar arquivos do projeto
O Claude diz que não encontra o arquivo ou retorna erro de permissão ao tentar editar
Causa

O VSCode foi aberto sem a pasta do projeto como workspace. O Claude só acessa arquivos dentro do workspace aberto.

Solução
Use File → Open Folder e abra a pasta raiz do projeto (ex: cco-fm-workspace)
Nunca abra apenas um arquivo avulso — sempre abra a pasta
O título da janela do VSCode deve mostrar o nome da pasta, não um arquivo
ℹ O Claude Code trabalha com o working directory do VSCode. Sem abrir a pasta correta, ele não tem contexto do projeto.
